---
title: "Listar Todos os Jobs (Admin)"
description: "Lista jobs de todas as contas com filtros — requer role admin"
api: "GET /admin/jobs"
---

Lista todos os jobs de todas as contas da plataforma, enriquecidos com dados da conta proprietária. Exclusivo para usuários com `role: admin`.

<Warning>
  Este endpoint expõe dados de todos os clientes. Use apenas para fins administrativos legítimos.
</Warning>

## Autenticação

Requer autenticação com **role admin** (cookie JWT ou API Key de usuário admin).

## Query Parameters

<ParamField query="status" type="string">
  Filtra por status do job. Valores válidos:
  `pending_approval`, `queued`, `processing`, `completed`, `failed`, `cancelled`.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Número da página.
</ParamField>

<ParamField query="per_page" type="integer" default="20">
  Itens por página. Máximo 100.
</ParamField>

## Resposta

<ResponseField name="data" type="array">
  Lista de jobs com dados completos + informações da conta.

  <Expandable title="Campos adicionais em relação ao endpoint de usuário">
    <ResponseField name="account_email" type="string">
      E-mail da conta proprietária do job.
    </ResponseField>
    <ResponseField name="account_name" type="string">
      Nome da conta proprietária.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  Informações de paginação padrão.
</ResponseField>

<RequestExample>

```bash cURL — jobs aguardando aprovação
curl "https://foneclear-web-production.up.railway.app/api/v1/admin/jobs?status=pending_approval" \
  -b cookies.txt
```

```bash cURL — todos os jobs
curl "https://foneclear-web-production.up.railway.app/api/v1/admin/jobs?per_page=50" \
  -b cookies.txt
```

</RequestExample>
