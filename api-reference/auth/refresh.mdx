---
title: "Renovar Token"
description: "Troca o refresh_token por um novo access_token sem precisar de novo login"
api: "POST /auth/refresh"
---

Usa o cookie `refresh_token` (válido por 7 dias) para emitir um novo `access_token` (válido por 15 minutos). Nenhum body é necessário — o token é lido automaticamente do cookie.

O cookie `refresh_token` tem path restrito a `/api/v1/auth/refresh`, ou seja, o browser só o envia para este endpoint.

## Autenticação

Requer o cookie `refresh_token`. Não requer `access_token` (que pode estar expirado).

## Resposta

<ResponseField name="message" type="string">
  Confirmação de renovação: `"Token renovado com sucesso."`
</ResponseField>

O cookie `access_token` é atualizado na resposta com um novo JWT de 15 minutos.

<RequestExample>

```bash cURL
curl -X POST https://foneclear-web-production.up.railway.app/api/v1/auth/refresh \
  -b cookies.txt \
  -c cookies.txt
```

```python Python
import httpx

with httpx.Client() as client:
    # Assume que o cliente tem os cookies salvos da sessão de login
    r = client.post(
        "https://foneclear-web-production.up.railway.app/api/v1/auth/refresh"
    )
    print(r.json())
    # Cookie access_token é atualizado automaticamente na sessão
```

</RequestExample>

<ResponseExample>

```json 200 OK
{
  "message": "Token renovado com sucesso."
}
```

```json 401 Refresh token ausente
{
  "error":   "unauthorized",
  "message": "Refresh token ausente."
}
```

```json 401 Refresh token expirado
{
  "error":   "unauthorized",
  "message": "Refresh token expirado."
}
```

```json 401 Token inválido
{
  "error":   "unauthorized",
  "message": "Refresh token inválido."
}
```

</ResponseExample>
